Thoughts on Progressive Enhancement

“Understanding Progressive Enhancement” by Aaron Gustafson was written in 2008. Back then, the mobile web was growing fast, but it was still considered an insignificant part of most website designs. Fully featured websites nowadays are expected to run javascript and stylesheets in two or three distinct “modes” depending on the viewing device. Much like there was a push by Steve Champeon, Nick Finck, and others to get developers to put accessibility and compatibility into their design process before the “last quarter of the development cycle,” the same push has come for mobile viewing considerations, and the progressive enhancement perspective on this may be interpreted as making peanut M&Ms alongside caramel M&Ms. The mobile devices may never see “just the caramel,” but it would be pulled into the DOM for certain devices.

Mozilla’s developer website has an entry on progressive enhancement, which usefully links two of the most integral tools in implementing it: feature detection and polyfills. Ironically, the entry on polyfill mostly describes ways that this obsolescence-friendly tool has had parts become obsolete. It also describes the first version of jQuery as an example of a polyfill.
